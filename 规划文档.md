# ai-bench 项目规划

## 项目定位

OpenClaw 专属压测工具 Skill。AI agent 通过聊天通道使用，给它一个 URL 和目标 QPS，直接出结果。
不需要理解线程数、ramp-up、pacing 等底层参数。

## 设计原则

- 输入就是三个参数：--url / --qps / --duration
- 输出是 Markdown 表格，AI 和人都能一眼看懂
- 单文件 bench.py，urllib3 或标准库降级
- 所有输出纯文本，适配 QQ / Discord / 终端等 channel

## 使用方式

```
用户/AI：帮我看下 /api/list 能抗多少，500 qps 跑 1 分钟
→ 调用 ai-bench --url http://host/api/list --qps 500 --duration 60
→ 输出 Markdown 报告，AI 直接解读
```

## 架构

```
CLI 解析 → Probe 探针 → Warmup 预热 → Worker Pool (Thread×N) + pacing
→ Collector 统计 → Markdown 报告输出
```
